### ℹ️ 문제 정보
  - 번호 : 1026
  - 난이도 : Silver
  - 분류 : 그리디, 정렬
    
<br />

### 🔎 문제 접근
와 B 두 배열이 주어질 때, B의 순서를 바꿔서 A와 B의 각 원소를 곱한 합의 최솟값을 구하는 문제다.
합을 최소로 만들려면 A의 가장 작은 수와 B의 가장 큰 수를 곱해야 한다는 그리디 전략을 사용했다.
A를 오름차순, B를 내림차순으로 정렬한 뒤 같은 인덱스끼리 곱해서 합산했다.

<br />

### 📝 사용한 자료구조
- ArrayList
- Collections.sort()
    
<br />

###  🦾 개선 포인트
- Scanner -> BufferedReader로 교체하여 입력 속도 개선
- ArrayList -> 배열로 교체
- B 배열을 따로 복사하지 않고 양 끝 인덱스 활용
    
<br />

###  💡 배운 점
합의 취소화를 위해 큰 수와 ㅈ가은 수 매칭하는 그리디 전략이 유효하며 정렬할때 람다식을 활용하면 간결하게 정렬이 가능함.
또한 배열을 새로 복사하지 않고 인덱스를 역방향으로 접근하면 불필요한 메모리 사용을 줄일 수 있음
